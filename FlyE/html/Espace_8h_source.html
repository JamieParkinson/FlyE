2  * Espace class for FlyE
3  *
4  * An Espace is essentially a container for the grid of e-field values in the accelerator. However, this grid is stored contiguously in field_ and Espace
5  * provides pseudomultidimensional access with at(), normAt() and magAt(). The e-field is treated as 4-dimensional: 3 dimensions for the position in space,
6  * and the final dimension as the direction of the field at the given point. The normField_ is maintained as the normalised field so we can safely scale field_.
7  *
8  * Espaces *must* be constructed with the params struct, which is defined here and contains various properties of the environment.
9  *
10  * The additional abilities of Espace are as follows:]
11  * gradientX(x,y,z), gradientY(x,y,z), gradientZ(x,y,z) - these functions get the field gradient at a given point.
12  * magAt(x,y,z) - this gets the field magnitude at a given point.
13  * normAt(x,y,z,d) accesses normField_, at(x,y,z,d) accesses field_
14  *
15  * Additionally, the compound addition/assignment operator ( += ) is overloaded for the class to add together e-fields as in matrix/tensor addition.
16  *
17  * Electrode is derived from Espace and offers methods specific to handling electrodes:
18  *
19  * import(n) imports the files generated by ExSimEck for electrode number n into normField_
20  * setVoltage(v) updates field_ with the values of normField_ multiplied by v
21  *
22  * Electrode also offers the static method sumElectrodes(allElectrodes, config, nToSum), which takes a vector of (shared_pointers to) Electrodes and sums nToSum of them
23  * into an Espace(config). It then returns this Espace with std::move to explicitly avoid a copy. If nToSum is not provided then all Electrodes in the vector are summed together.
24  * This summation is achieved using the aforementioned compound assignment overload, which gives good readability but is slow if the compilation is not optimised (if it is optimised
25  * then the operation is expanded inline).
26  *
27  * Jamie Parkinson, July 2014
28  *
